Test plan

Run these before going live.

Use this page to test the customer path from request form to backend row to finished deliverable handoff.

Minimum live test

Run two complete test orders.

One normal customer package and one Shop / CNC planning package are enough to prove the system works before public launch.

H38-T001

Problem Snapshot test

  • Fake customer: homeowner/shop owner with messy garage photos.
  • Input: short problem description, 3–5 photos or placeholder notes, desired outcome.
  • Backend route: Lead Intake → Test Orders → Product Build Sheets → QA Handoff.
  • Deliverable: one-page summary, first five actions, missing-info list, risk notes.
  • Acceptance: customer can take one clear action without a phone call.
H38-A001

Automation Opportunity Snapshot test

  • Fake customer: small shop with machine idle or repetitive handling issue.
  • Input: machine/process photos or description, cycle-time guess, operator pain, output goal.
  • Backend route: Lead Intake → Automation Test Orders → Automation Products → QA Handoff.
  • Deliverable: opportunity memo, missing data list, stop/go/collect-more-data recommendation.
  • Acceptance: owner knows whether to collect data, ask vendor questions, or stop.
Full package test matrix

Use this after the minimum live test.

Each package should eventually have one completed sample deliverable.

H38-T002

Project Packet

  • Input: rough layout dimensions, goal, product links, photos.
  • Output: zone plan, material groups, phased build order, checklist.
  • QA: assumptions and missing measurements clearly named.
H38-T003

Shop Flow Review

  • Input: workspace photos, tool list, repeated jobs, pain points.
  • Output: zones, staging rules, first five fixes.
  • QA: first move is clear and low-risk.
H38-T004

Business Cleanup

  • Input: service list, quote examples, text screenshots.
  • Output: service menu, lead tracker, quote/deposit path, follow-up scripts.
  • QA: every lead has status, amount, next action, follow-up date.
H38-T005

Digital setup

  • Input: services, photos, current link/profile, desired form fields.
  • Output: page outline, form map, tracker columns, prompt pack.
  • QA: visitor path is page → form → tracker → reply.
H38-T006

Cleanup Rescue

  • Input: messy file/folder notes, receipts, screenshots, dates.
  • Output: folder map, naming rule, cleanup index, action list.
  • QA: files have place, purpose, status, next action.
H38-T007

custom work

  • Input: repeated problem, desired output, sample rows/screenshots.
  • Output: version-one scope, test rows, prototype map, handoff SOP.
  • QA: boundaries and not-included items are named.

Test request script

  1. Open the live form.
  2. Submit one fake request using the test ID in the description field.
  3. Confirm owner alert email arrives.
  4. Confirm customer confirmation email arrives.
  5. Open backend workbook and find the new lead row.
  6. Create or mark the matching test order.
  7. Build the deliverable using the matching build sheet.
  8. Run QA before delivery.
  9. Send a test handoff email to yourself.
  10. Save the finished test output as the public sample/proof item.

Test before sharing the URL.

One successful form submission plus one finished deliverable proves the system is ready enough to launch.
