Repeatable Demo Product System

One demo structure works for every active product.

The demo system uses the same four-step proof path for every product: choose the product, apply the sample input, display the finished output, then stop at the approval gate.

1. Product selectedEach product has a card, ID, description, and “Try sample run” anchor.
2. Sample input usedSample Garage Bay input is reused so a tester can compare outputs side by side.
3. Output renderedEach finished example shows the messy input → measurements → CAD/layout/storage visual plus product-specific result.
4. Approval gateNo instant delivery. Rick Review Required / Owner Approval Required.
Repeatable rule: every future non-custom demo product must use the same structure: product card → sample input → finished output → proof trail → approval gate.
